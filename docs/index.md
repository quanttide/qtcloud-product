# 量潮产品云

## 场景

产品团队在做一个产品，但没有人能直接看到"这个产品长什么样"。产品全景图以隐性知识的形式存在于人的脑子里——PM 知道一部分，工程师知道一部分，但拼不出完整的地图。

## 困境

隐性知识是产品的常态。新人理解一个产品靠访谈、靠读文档、靠自己脑补，这是一个把散落的碎片拼成全景图的过程。更根本的是，**没有人能指着同一个地方说"这个功能在这条链路上"**，因为没有那条链路——链路只在脑子里。

## 救援

量潮产品云把产品的隐性结构显式化。以用户故事地图为骨架，将用户活动 → 用户任务 → 用户故事变成所有人可见的共享地图——不是一份文档，是产品的结构本身。

## 新生

团队建立了共享的产品全景图意识。讨论产品时指向的是地图上的同一个位置。新的子产品不是从空白开始，而是从已有地图的接缝处长出新分支。隐性知识变成了公共知识。
