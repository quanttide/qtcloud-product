# 展示用户故事

有两种用户故事的格式可以选择。

## 方案1：用户故事地图提供的3C

包括：

- Card
- Conversation
- Confirmation

参考格式：

- https://medium.com/@mojamcpds/card-conversation-confirmation-d88c074dd58c

## 方案2：用户故事资料提供的格式

包括：

- Title
- Description
- Acceptance Criteria

参考格式：

- https://www.youtube.com/watch?v=LEPLaYcdgeg

## 对比

主要的分歧在于使用对话还是富文本说明详细产品需求。

前者比较适合在头脑风暴中记录交谈，后者比较适合产品策划写正式需求文档。

如果两者都兼容，则产品设计比较麻烦。
